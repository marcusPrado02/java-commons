# Guia de Migração: v0.1.x → v1.0.0

Este guia descreve as mudanças planejadas para a primeira release estável do java-commons
e o caminho de migração para quem já usa snapshots `0.1.x`.

> **Status**: v1.0.0 em desenvolvimento. Mudanças abaixo refletem o estado atual do `main`
> e os itens marcados com ⚠️ são potencialmente breaking.

---

## Visão Geral das Mudanças

| Área | Tipo | Impacto |
|------|------|---------|
| `Result<T>` — novos métodos assíncronos | Adição | Nenhum (backward compatible) |
| `OutboxProcessor.processBatch(int)` | Adição | Nenhum (backward compatible) |
| `BackupService` — differential + point-in-time | Adição | Nenhum (backward compatible) |
| `DomainEvent` interface | Revisão ⚠️ | Implementações existentes precisam adicionar `aggregateVersion()` e `metadata()` |
| `commons-spring-starter-*` artifact IDs | Renomeação ⚠️ | Atualizar dependências no `pom.xml` |
| `commons-testkit` → `commons-testkit-core` | Renomeação ⚠️ | Atualizar `<scope>test</scope>` |
| Cobertura JaCoCo como gate obrigatório | Novo gate | Build falha se cobertura < 60% linha / 55% branch |
| PMD / SpotBugs / Checkstyle obrigatórios | Novo gate | Build falha em violações críticas |

## Mudanças Breaking

### 1. `DomainEvent` — novos métodos obrigatórios

**O que mudou**: A interface `DomainEvent` em `commons-kernel-ddd` agora exige dois métodos
adicionais para suportar event versioning e metadata propagation.

**Antes (v0.1.x)**:
```java
public class OrderCreated implements DomainEvent {
    @Override public EventId eventId()       { return EventId.newId(); }
    @Override public Instant occurredAt()    { return Instant.now(); }
    @Override public String aggregateType()  { return "Order"; }
    @Override public String aggregateId()    { return orderId; }
}
```

**Depois (v1.0.0)**:
```java
public class OrderCreated implements DomainEvent {
    @Override public EventId eventId()          { return EventId.newId(); }
    @Override public Instant occurredAt()       { return Instant.now(); }
    @Override public String aggregateType()     { return "Order"; }
    @Override public String aggregateId()       { return orderId; }
    @Override public long aggregateVersion()    { return 1L; }          // NOVO
    @Override public EventMetadata metadata()   { return EventMetadata.empty(); } // NOVO
}
```

**Migração automática** (encontre todas as implementações):
```bash
grep -r "implements DomainEvent" src/ --include="*.java"
```

Adicione os dois métodos acima. Para a maioria dos casos `1L` e `EventMetadata.empty()` são adequados.

---

### 2. Artifact IDs dos Spring Starters renomeados

**O que mudou**: Starters prefixados com `commons-spring-` foram renomeados para
`commons-spring-starter-` para maior clareza.

| Antes (v0.1.x) | Depois (v1.0.0) |
|----------------|-----------------|
| `commons-spring-observability` | `commons-spring-starter-observability` |
| `commons-spring-resilience` | `commons-spring-starter-resilience` |
| `commons-spring-outbox` | `commons-spring-starter-outbox` |
| `commons-spring-idempotency` | `commons-spring-starter-idempotency` |
| `commons-spring-cache` | `commons-spring-starter-cache` |
| `commons-spring-otel` | `commons-spring-starter-otel` |
| `commons-spring-secrets` | `commons-spring-starter-secrets` |

**Migração**:
```bash
# Encontre e substitua no pom.xml
sed -i 's/<artifactId>commons-spring-/<artifactId>commons-spring-starter-/g' pom.xml
```

---

### 3. `commons-testkit` → `commons-testkit-core`

**O que mudou**: O módulo de test utilities foi dividido em:
- `commons-testkit-core` — utilitários genéricos de teste
- `commons-testkit-contracts` — testes de contrato (Pact/Spring Cloud Contract)
- `commons-testkit-archunit` — regras ArchUnit reutilizáveis

**Migração** no `pom.xml`:
```xml
<!-- Antes -->
<dependency>
    <groupId>com.marcusprado02.commons</groupId>
    <artifactId>commons-testkit</artifactId>
    <scope>test</scope>
</dependency>

<!-- Depois -->
<dependency>
    <groupId>com.marcusprado02.commons</groupId>
    <artifactId>commons-testkit-core</artifactId>
    <scope>test</scope>
</dependency>
```

### 4. Gates de qualidade obrigatórios

A partir de v1.0.0, o build **falha** nas seguintes condições:

| Gate | Threshold | Como corrigir |
|------|-----------|--------------|
| JaCoCo (linha) | < 60% | Adicionar testes unitários |
| JaCoCo (branch) | < 55% | Cobrir branches condicionais |
| SpotBugs | Qualquer bug MEDIUM+ | Corrigir problemas reportados |
| Checkstyle | Qualquer violação | Ajustar formatação/estilo |
| PMD | Qualquer violação crítica | Refatorar código problemático |
| OWASP CVE | CVSS ≥ 9 | Atualizar dependência vulnerável |

Para verificar localmente antes de commitar:
```bash
./mvnw verify -pl <seu-módulo>
```

Para pular gates em emergência (não recomendado):
```bash
./mvnw verify -Dspotbugs.skip=true -Dcheckstyle.skip=true -Dpmd.skip=true -Djacoco.skip=true
```

## Mudanças não-breaking (novas funcionalidades)

### `Result<T>` — operações assíncronas

```java
// Novo em v0.1.x (adicionado progressivamente)
CompletableFuture<Result<Order>> future = result
    .mapAsync(order -> orderRepo.saveAsync(order))
    .flatMapAsync(order -> notifyAsync(order));

// toFuture()
CompletableFuture<Order> cf = result.toFuture(); // lança DomainException se fail
```

### `OutboxProcessor.processBatch(int)`

```java
// Processa exatamente N mensagens pendentes
outboxProcessor.processBatch(50);
```

### `BackupService` — novas operações

```java
// Backup diferencial (mudanças desde o último full)
service.createDifferentialBackup("Diff Backup", fullBackupId, config);

// Point-in-time restore
service.restorePointInTime("2026-03-01T12:00:00Z", restoreConfig);
```

## Roteiro de Migração

### Passo 1 — Atualizar BOM

```xml
<dependency>
    <groupId>com.marcusprado02.commons</groupId>
    <artifactId>commons-bom</artifactId>
    <version>1.0.0</version>
    <type>pom</type>
    <scope>import</scope>
</dependency>
```

### Passo 2 — Compilar e identificar erros

```bash
./mvnw clean compile 2>&1 | grep -E "error:|ERROR"
```

Erros esperados:
- `cannot find symbol: method aggregateVersion()` → adicionar método
- `cannot find symbol: method metadata()` → adicionar método
- `package ... does not exist` para starters → atualizar artifact IDs

### Passo 3 — Rodar os gates de qualidade

```bash
./mvnw verify
```

Corrija quaisquer falhas de SpotBugs, PMD, Checkstyle ou cobertura.

### Passo 4 — Executar testes de integração

```bash
./mvnw verify -P integration-tests
```

### Passo 5 — Validar no staging

Implante em staging e execute o smoke test:
```bash
# Via GitHub Actions
gh workflow run smoke-test.yml
```

---

## Rollback

Se precisar reverter para v0.1.x:

```xml
<commons.version>0.1.0-SNAPSHOT</commons.version>
```

```bash
./mvnw clean install -U
```
